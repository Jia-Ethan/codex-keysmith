# 安全政策 / Security Policy

## 私密报告漏洞

请不要通过公开 issue 报告安全漏洞。使用 GitHub 的 [Private vulnerability reporting](https://github.com/Jia-Ethan/codex-keysmith/security/advisories/new) 提交私密报告。

报告中请尽量包含：

- 受影响的 commit 或版本；
- 操作系统、Python 版本和 Codex CLI 版本；
- 可复现的最小步骤；
- 影响范围，以及已知的缓解方式。

提交前请删除 token、cookie、用户名、私人路径和完整配置内容。项目维护者将在 GitHub Security Advisories 中跟进报告；本仓库不承诺固定响应时限。

本项目当前不维护独立的历史版本分支，安全修复以默认分支的最新代码为准。

---

## Private vulnerability reporting

Do not report a vulnerability through a public issue. Submit it through GitHub [Private vulnerability reporting](https://github.com/Jia-Ethan/codex-keysmith/security/advisories/new).

Include the affected commit or version, operating system, Python version, Codex CLI version, minimal reproduction steps, impact, and any known mitigation. Remove tokens, cookies, usernames, private paths, and complete configuration files before submitting.

Maintainers will follow up through GitHub Security Advisories. This repository does not promise a fixed response time. Historical release branches are not maintained; security fixes target the latest code on the default branch.
